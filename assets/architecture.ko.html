패키지 단위 · 선제적

설치 시점 레인 (Install-time)

모든 에이전트 의존성 설치를 게이트한다. 3단계: advisory check 가 approved-spec ledger 를 쓰고, 빠른 PreToolUse 커맨드 가드가 막고, npm PostToolUse effect gate(진짜 권위)가 어긋난 건 reorg(롤백)한다.

레포 단위 · opt-in

릴리스 시점 레인 (Release-time)

레포 트리 위생을 한 namespace 로: secret 스캔, 멀티-ecosystem 의존성 audit, 레포 git-hook 설치/검사, 자세 진단 doctor. safedeps 는 실행을, 레포는 정책을 소유한다.

설치 파이프라인 — 전엔 게이트, 후엔 집행
advisory
PHASE 1

advisory 검사

safedeps check
  • OSV → KEV → GHSA 조회
  • npm: 스크립트 없는 임시 lockfile 로 전체 closure 해석, OSV /v1/querybatch
  • clean/patched → approved-spec 를 ledger 에 기록 (+ transitive_specs)
advisory
PHASE 2

빠른 커맨드 가드

PreToolUse hook
  • 설치 명령 파싱, 명백한 미승인 설치 차단
  • lockfile / manifest 스냅샷
  • Claude: --ignore-scripts 주입 → 설치가 무실행(inert)으로
authority
PHASE 3

npm effect gate

PostToolUse hook
  • 실제 package-lock.json closure 를 읽음
  • 모든 pkg@version 을 ledger + transitive_specs 와 대조, OSV 배치 재조회
  • closure 가 clean 검증된 뒤에만 npm rebuild
clean → 유지 어긋남 → reorg KEV → hard block
▸ 커맨드 가드는 best-effort UX 레이어 ▸ effect gate 는 커맨드-독립: 파서 맹점이어도 live closure 검사는 돈다
advisory 출처 & 엔진 비대칭

불변식 — 깨지 말 것
effect 가 권위
npm 은 PostToolUse closure 검사가 결정한다 — 커맨드-텍스트 가드가 아니라.
No silent fallback
provider 미스는 fail-closed. 모든 bypass 는 관측 가능하고 로깅된다.
의존성 0
도구 자체가 npm deps 0 — 실수가 아니라 보안 속성.
inert install (Claude)
거부된 패키지는 lifecycle 스크립트를 절대 실행하지 않는다.
OSV 는 단수
KEV 는 overlay, GHSA 는 enrichment — 두 번째 동등 진실 없음.
ledger ≠ 경계
동일 사용자 편의 캐시일 뿐, 동일 사용자 공격자 방벽이 아니다 (서명 전까지).
