safedeps architecture

A local, zero-dependency security gate for development dependency installs. Approval happens before the install against canonical advisory evidence; enforcement happens after, against what actually landed on disk.
v2.9.0 · Claude Code + Codex CLI
OSV single canonical truth No silent fallback · fail-closed Zero npm dependencies No SaaS — local CLI + public DBs Effect-primary = npm-only
Two lanes
per-package · proactive

Install-time lane

Gates every agent dependency install. Three phases: an advisory check that writes an approved-spec ledger, a fast PreToolUse command guard, and the npm PostToolUse effect gate — the real authority — that reorgs (rolls back) anything that diverges.

per-repo · opt-in

Release-time lane

Repo-tree hygiene exposed under one namespace: secret scan, multi-ecosystem dependency audit, repo git-hook install/check, and a posture doctor. safedeps owns execution; the repo owns policy.

PHASE 1

Advisory check

safedeps check
  • Query OSV → KEV → GHSA
  • npm: build a script-free temp lockfile, resolve the full closure, OSV /v1/querybatch
  • Clean/patched → write approved-spec to the ledger (+ transitive_specs)
advisory
PHASE 2

Fast command gate

PreToolUse hook
  • Parse the install command; block obvious unapproved installs
  • Snapshot lockfiles / manifests
  • Claude: inject --ignore-scripts → install runs inert
authority
PHASE 3

npm effect gate

PostToolUse hook
  • Read the actual package-lock.json closure
  • Verify every pkg@version vs ledger + transitive_specs; re-query OSV in batch
  • npm rebuild only after the closure verifies clean
clean → keep diverged → reorg KEV → hard block
▸ command guard is a best-effort UX layer ▸ the effect gate is command-independent: a parser blind spot still gets the live closure check

Invariants — do not break
Effect is the authority
For npm, the PostToolUse closure check decides — not the command-text guard.
No silent fallback
A provider miss is fail-closed; every bypass is observable and logged.
Zero dependencies
The tool itself ships no npm deps — a security property, not an oversight.
Inert install (Claude)
Rejected packages never execute their lifecycle scripts.
OSV is singular
KEV is an overlay, GHSA is enrichment — no second co-equal truth.
Ledger ≠ boundary
A same-user convenience cache, not a wall against a same-user attacker (until signing).
